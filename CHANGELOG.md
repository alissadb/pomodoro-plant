## How Releases Work

This project uses automated releases:
- Every PR merge to `main` triggers a new release
- Version bumping is based on PR labels (feature, fix, docs, etc.)
- Release notes are auto-generated with emoji prefixes
- Currently in 0.x.x (pre-1.0) version range

### Version Bumping

- **MINOR** (0.0.1 → 0.1.0): `feature`, `enhancement`, or `breaking` labels
- **PATCH** (0.0.1 → 0.0.2): `fix`, `docs`, `chore`, `test`, etc.
